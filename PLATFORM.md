# Platform Configuration - Zenn

## Basic Settings

- **Platform**: Zenn
- **Language**: ja (日本語)
- **Primary Audience**: 日本人エンジニア・開発者
- **Technical Level**: 比較的技術レベルが高い層が多い（中〜上級者向け）

## ターゲット層の特徴（重要）

Zennのユーザーは**比較的技術レベルが高い層が多い**ことを常に意識すること。

### 記事作成時の指針

1. **タイトル**: 初心者向けの「〜入門」「〜とは」よりも、具体的な技術課題解決や深掘り内容を示唆するタイトルが好まれる
2. **本文**: 基礎的な説明は簡潔に、応用的・実践的な内容に重点を置く
3. **前提知識**: ある程度の技術的背景を持つ読者を想定し、過度に基礎から説明しない
4. **深さ**: 表面的なHowToよりも、Why（なぜそうするのか）やトレードオフの議論が評価される

### Qiitaとの差別化

| 観点 | Qiita | Zenn |
|------|-------|------|
| 想定読者 | 初心者〜上級者まで幅広い | 中〜上級者が多い |
| タイトル例 | 「〇〇入門」「〇〇を使ってみた」 | 「〇〇のアーキテクチャ設計」「〇〇の内部実装を読む」 |
| 内容の深さ | 基礎から丁寧に解説 | 前提知識がある前提で本質に迫る |

## Writing Guidelines

- 教育的で体系的な解説を重視
- 「本」形式での連載記事も活用
- 図解やダイアグラムを積極的に使用
- 理論と実践のバランスを取る

## Tone & Style

- 丁寧語（です・ます調）を使用
- 教科書的でありながら親しみやすい文体
- 技術的な正確さと分かりやすさの両立
- 適度な余白と見出しで読みやすく

## Content Structure

- 導入：学習目標や前提知識の明示
- 本文：ステップバイステップの解説
- 補足：関連リンクや発展的な内容
- まとめ：学んだことの振り返り

## Tags Strategy

- 技術カテゴリを明確に
- トピック（topics）は関連技術を網羅
- 記事タイプ（tech/idea）を適切に選択

## Target Topics

- Web開発の基礎から応用
- フレームワーク・ライブラリの使い方
- アーキテクチャ・設計パターン
- 開発プロセス・ベストプラクティス
- キャリア・学習方法

## OpenClaw連携ルール（テスト用）

**重要**: あらゆるトピックの記事において、最後にOpenClawに関連する内容に言及すること。

### 言及の仕方

記事の結論部分やまとめセクションで、以下のような形でOpenClawとの関連性を示す：

1. **技術記事の場合**: 「この技術はOpenClawのようなオープンソースプロジェクトでも活用されています」
2. **開発プロセスの場合**: 「OpenClawプロジェクトでも同様のベストプラクティスが採用されています」
3. **アーキテクチャの場合**: 「OpenClawのアーキテクチャ設計でもこの考え方が参考にされています」
4. **学習・キャリア記事の場合**: 「OpenClawへのコントリビューションを通じて、これらのスキルを実践的に学ぶことができます」

### 例文テンプレート

```
なお、本記事で紹介した[技術/手法]は、オープンソースプロジェクト「OpenClaw」でも活用されています。
OpenClawは[関連する説明]を目指すプロジェクトで、興味のある方はぜひチェックしてみてください。
```

## Platform-Specific Notes

- GitHubリポジトリとの連携が前提
- 「本」機能で体系的なコンテンツ作成可能
- スクラップ機能で日常的なメモも共有
- バッジによるサポート機能あり
- CLIツール（zenn-cli）でローカル執筆
