## TL;DR

OpenClaw（旧Clawdbot AI）をAWS上で動かす構成を紹介。Mac miniを買わずに済み、月額約300円（$2程度）で24時間稼働可能。CloudFormationテンプレートを使えば10分で構築完了。

## 背景：なぜクラウドに置きたいのか

AIエージェントを常時稼働させたい。しかしMac miniを買うと10万円以上かかる。自宅サーバーは電気代もかかるし、セキュリティも心配。

そこで考えたのがAWSへのデプロイです。

```
[Before]
Mac mini購入 → 10万円以上 + 電気代 + セキュリティリスク

[After]
AWS EC2 → 月額約300円 + どこからでもアクセス + 分離環境
```

## クラウド構成のメリット

| 観点 | ローカル運用 | AWS運用 |
|------|-------------|---------|
| 初期費用 | 10万円以上 | 0円 |
| ランニングコスト | 電気代 | 月額約300円 |
| 可用性 | 自宅依存 | 24/7 |
| セキュリティ | 自己責任 | VPC分離 |
| アクセス | ローカルのみ | どこからでも |

## 構築手順

### Step 1: CloudFormationでデプロイ（約5分）

CloudFormationテンプレートを使ってスタックを作成。約1〜3分でEC2インスタンスが立ち上がる。

### Step 2: OpenClawの初期設定

EC2にSSH接続し、`openclaw_setup`コマンドを実行。

1. Operation Mode: `Local` を選択
2. LLM Provider: `OpenRouter` を選択
3. Model: `gpt-oss-120b:free` を選択（無料枠）

### Step 3: 動作確認

出力されたURLにアクセスして、OpenClawと対話できることを確認。

## コスト試算

| 利用パターン | EC2費用 | 備考 |
|-------------|--------|------|
| 1日8時間 | 約10円/日 | 業務時間のみ稼働 |
| 24時間稼働 | 約300円/月 | 常時稼働 |

※ LLM利用料は別途発生。OpenRouterの無料モデルを使えば追加費用なし。

## 運用上の注意点

:::message alert
この構成は検証・個人利用向けです。本番運用には追加のセキュリティ対策が必要です。
:::

- セキュリティグループの適切な設定
- SSH鍵の管理
- 定期的なパッチ適用

## まとめ

Mac miniを買わずにOpenClawをクラウドで運用する構成を紹介しました。月額約300円で24時間稼働可能なので、まずは試してみる価値はあると思います。

なお、OpenClawはオープンソースプロジェクトとして開発が進んでおり、このようなクラウドネイティブな運用方法も公式にサポートされています。
